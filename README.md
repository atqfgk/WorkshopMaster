# WorkshopMaster — Автоматизация столярной мастерской

### 1. Описание проекта
Информационная система для управления заказами, контроля этапов производства, учёта материалов и расчёта себестоимости в столярной мастерской «Деревянный декор».

### 2. Цель проекта
Устранить ручной учёт (бумажные наряды, Excel), сократить срывы сроков, повысить прозрачность загрузки сотрудников и точность финансовых расчётов.

### 3. Технологический стек
- **Язык / платформа:** C#, WPF (.NET 6)
- **База данных:** MS SQL Server
- **Доступ к данным:** ADO.NET (оперативные операции), Entity Framework Core (отчёты)

### 4. Основные возможности (по ролям)

| Роль | Функции |
|------|---------|
| **Менеджер** | Регистрация клиентов, создание заказов, добавление этапов/материалов, расчёт предварительной стоимости, утверждение заказа. |
| **Дизайнер** | Декомпозиция заказа на этапы, определение потребности в материалах. |
| **Мастер** | Просмотр назначенных этапов, отметка начала/окончания, списание материалов. |
| **Кладовщик** | Учёт прихода материалов, контроль остатков, управление справочником материалов. |
| **Руководитель** | Полный доступ, аналитические отчёты (статусы заказов, рентабельность, загрузка сотрудников). |

### 5. Структура базы данных (12 таблиц)

- `Клиенты`, `Сотрудники`, `Материалы`, `Поставщики`, `ЭтапыРабот`
- `Заказы`, `ПлановыеЭтапыЗаказа`, `ФактическиеЭтапыЗаказа`
- `СписанияМатериалов`, `Платежи`, `ПоступленияМатериалов`, `ПозицииПоступления`

**Ключевые связи:**  
Заказы → Плановые этапы → Сотрудники / Этапы работ;  
Списания → Материалы;  
Поступления → Поставщики → Материалы.

**Дополнительные объекты БД:**  
- Представление `vw_СтатусЗаказов` (отклонения по срокам)  
- Хранимая процедура `usp_РассчитатьСебестоимостьЗаказа`  
- Триггер для автоматического уменьшения остатков материалов при списании
